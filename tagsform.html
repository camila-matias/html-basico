Formulários HTML - Principais tags

Tags em formulários HTML

form
É a tag que inicializa e finaliza um formulário interativo, englobando todo seu conteúdo. Possui os atributos específicos:
  • action, que define uma url, relativa ou absoluta, para onde os dados do formulário irão após submissão deste.
  • method, que define o método HTTP utilizado quando o formulário é submetido (get, post, entre outros).
  • target, que define se o url indicada na action será aberta na mesma aba do navegador, em uma aba ou janela diferentes.
  • autocomplete, que define se o navegador poderá utilizar o recurso de auto-completar respostas ou não.
  • novalidate, que indica que o navegador não irá validar o formulário em sua submissão (validação do lado do cliente).
fieldset
Tag que agrupa campos de um mesmo tema. Em geral, agrupa campos do tipo radio ou checkbox que possuem um tema em comum.
legend
Tag filha da tag fieldset, é utilizada como legenda para o agrupamento de campos, ou seja, sobre o que é a tag fieldset a qual está associada.
label
Tag utilizada como legenda para as tags do tipo input. Possuem o atributo específico for, em que é indicado o nome do input ao qual a label está associado.
input
É a tag de campo de formulário. Possui vários tipos, cada um com seus atributos gerais e específicos. Os tipos mais comuns são:
  • text, que é um campo de texto, podendo ter atributos como minlength e maxlength, que indicam a quantidade mínima e máxima de caracteres permitidos no texto.
  • date, que é um campo de data.
  • email, que é um campo de e-mail. Atualmente este campo é automaticamente validado pelo navegador na submissão do formulário - exceto quando o novalidate é utilizado - para validar se os valores preenchidos possuem o padrão correto para e-mails.
  • radio, define uma caixa de seleção única, ou seja, caso mais de um input radio seja utilizado para seleções dentro de um mesmo tema, apenas um input poderá ser marcado. Possui o atributo value para que seja indicado o valor que cada opção indica dentro do formulário.
  • tel, que indica que os valores preenchidos no campo serão números de telefone.
  • url, que é um campo de url e, assim como o campo de e-mail, é automaticamente validado pelo navegador com o padrão de urls.
  • file, que é um campo de inserção de documentos no preenchimento do formulário.
  • textarea, que é um campo de caixa de texto, porém com a possibilidade de aumentar o tamanho de linhas e colunas com os atributos específicos rows e cols.
  • password, que é um campo para senhas, em que a visualização do preenchimento é restrita e não se pode verificar visualmente o que foi preenchido.
  • checkbox, que é um campo de caixa de seleção múltipla, em que múltiplos checkboxes de um mesmo tema podem ser marcados.
  • number, que é um campo de números.
  • range, que é um campo de range, podendo ter valores máximos e mínimos definidos com os atributos específicos max e min.
  • image, que é um tipo em que a imagem utilizada como fonte pode substituir o botão de submissão do formulário.
  • color, que abre uma caixa de seleção de cores.
  • submit, que é um tipo de input que se comporta como um botão, podendo ser o botão de submissão do formulário.
  • reset, um tipo que, assim como o submit, se comporta como um botão. Clicando em um input do tipo reset, o formulário é resetado em todos os seus campos.
select
Esta tag abre uma lista para seleção de dados, sendo de seleção única.
option
Tag filha da tag select, é o campo de opção da lista de seleção criada pela tag mãe. Possui o atributo value para que seja indicado o valor que aquela opção indica dentro da lista e do formulário.
button
Esta tag cria um botão sem comportamento específico default, mas que pode ser utilizado para submissão de formulários caso seja do tipo submit.

Atributos gerais

name
É o identificador do campo dentro do formulário e na sua submissão. É por onde um possível backend identifica o valor preenchido dentro do campo.
id
É o identificador do campo dentro do código e utilizado em possíveis manipulações do formulário utilizando Javascript.
placeholder
É um texto dentro do campo que é utilizado como referência do que deve ser preenchido no campo em específico. Assim que o usuário começa a preencher o campo, o placeholder é apagado.

Informações sobre validação

Validação do lado do cliente

A validação do lado do cliente é realizada nos formulários por meio da tipagem dos inputs (como url ou e-mail) em que o navegador confronta o valor preenchido com um padrão especificado e por meio do atributo required, em que o navegador irá avaliar se os campos marcados com esse atributo estão preenchidos. Caso alguma das validações resulte em não correspondência ou não preenchimento, o navegador impedirá o formulário de ser submetido.

O atributo pattern

Input dos tipos text, tel, email, url, password e search podem ser contemplados com o atributo pattern, que define um padrão específico com o qual os valores de preenchimento serão validados.

O padrão definido deverá ser uma expressão regular Javascript válida (comumente chamadas de RegEx) e, por questões de acessibilidade, é recomendado que, acompanhado do atributo pattern, venha o atributo title, que dá mais informações sobre o padrão que está sendo utilizado.

Validações com Javascript

Outros tipos de validação, como igualdade entre campos de "senha" e "confirmação de senha", são realizados por meio de Javascript e não serão abordados neste documento.
